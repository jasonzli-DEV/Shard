**Goal:** Build Shard â€” self-hosted, MongoDB-backed cloud storage with a Google-Drive-like UI, per-user Atlas M0 autoscaling, OAuth auth, a REST API, a branded setup wizard â€” fully tested and deployed to `pi.local` replacing D-Drive.

**Architecture:** React+Vite+TS frontend â†’ Node/Express+TS backend â†’ Mongoose on an operator "starter" cluster for all metadata; file bytes stored in GridFS across a per-user fleet of free Atlas M0 clusters auto-provisioned via the Atlas Admin API (digest auth) at 80% of 512MB. Ported autoscaler pattern from the Aura bot.

**Tech Stack:** TypeScript, Node 20, Express, Mongoose 8, MongoDB GridFS, React 18, Vite, React Router, Passport (google/github OAuth) or arctic, JWT, Jest/Vitest, Playwright, Docker Compose.

## Global Constraints

- All persistence is MongoDB. No Postgres, no Prisma. Mongoose everywhere.
- Starter cluster stores metadata only; file bytes go to per-user fleet GridFS.
- Each user supplies their own Atlas org API key pairs (multiple per user). Storage autoscales inside their orgs.
- Connection URIs and Atlas private keys stored **plaintext** in the starter DB (intentional â€” keeps setup trivial).
- Atlas: M0 / TENANT / AWS / US_EAST_1, REPLICASET, accept header `application/vnd.atlas.2023-02-01+json`, base `https://cloud.mongodb.com/api/atlas/v2`, digest auth with org public/private key.
- Autoscale: 512MB limit, 0.80 provision threshold, 0.75 warn, 10-min check, 60s keepalive. Org cap ~250 clusters â†’ roll to user's next org key.
- Brand name "Shard". Setup wizard must be hand-designed, not a generic centered-card AI template.
- Production-ready: no stubs/TODOs/placeholders in shipped code. Unit + integration + E2E tests must pass.
- Deploy: Docker Compose to `jasonzli@pi.local`, replacing D-Drive (back it up first).
- Reference implementations to port/adapt: `/Users/zhixiangli/Github/Aura/src/db/atlas.js` (Atlas client) and `/Users/zhixiangli/Github/Aura/src/db/connection.js` (cluster manager). Reference D-Drive for UI/route patterns: `/Users/zhixiangli/Github/D-Drive/backend/src` and `/frontend`.
